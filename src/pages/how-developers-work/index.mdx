<Level id="using-advanced-features" number="5" name="Using advanced features" difficulty="Nightmare!" objectives={<span>Learn <code>git</code> internals and advanced GitHub features. Well, basically, survive.</span>} />

1. <div>Exclude <strong>.DS_Store</strong> from the repository by either adding it to <strong>.gitignore</strong> <sup><strong><a href="https://git-scm.com/docs/gitignore" title="gitignore">[1]</a><a href="https://github.com/github/gitignore" title="github/gitignore">[2]</a></strong></sup> or by setting an <code>export-ignore</code> attribute for it in <strong>.gitattributes</strong> <sup><strong><a href="https://git-scm.com/docs/gitattributes" title="gitattributes">[1]</a><a href="https://github.com/alexkaratarakis/gitattributes" title="alexkaratarakis/gitattributes">[2]</a></strong></sup></div>
2. <p>Add <strong>volodymyrkushnir.com</strong> as a submodule by running <code>git submodule add git@github.com:volodymyr-kushnir/volodymyrkushnir.com.git dependencies/volodymyrkushnir.com</code> <sup><strong><a href="https://git-scm.com/docs/git-submodule" title="git-submodule">[1]</a></strong></sup></p>
   
   <div class="bubble small subtle">
     If you run into errors ‚Äî try to use <strong>https://</strong> endpoint. Also, keep in mind that <a href="https://stackoverflow.com/questions/1260748/how-do-i-remove-a-submodule/7646931"><strong>removing a submodule</strong></a> is somewhat a tricky process
   </div>

3. <div>Use <strong>pre-commit</strong> git hook <sup><strong><a href="https://git-scm.com/docs/githooks" title="githooks">[1]</a></strong></sup> to generate new <strong>Service Worker</strong> based on <a href="https://developers.google.com/web/tools/workbox/" title="Workbox"><strong>Workbox</strong></a> before each commit</div>
   
   1. Run `npm install --global workbox-cli` to install Workbox globally
   2. Create **workbox-config.js** with the following content:
      ```json
      module.exports = {
        "globDirectory": "./",
        "globPatterns": [
          "**/*.{png,svg,ico,jpg,mp3,css,html}"
        ],
        "swDest": "sw.js"
      };
      ```
   3. Create **.git/hooks/pre-commit** file that should look roughly like this:
      ```bash
      #!/bin/sh
      if workbox generateSW workbox-config.js ; then
        git add sw.js
        exit 0
      else
        echo "Cannot generate sw.js"
        echo "Aborting"
      fi
      ```
      <div class="bubble small subtle">
        Make sure it's executable, <code>chmod +x .git/hooks/pre-commit</code> if necessary. Keep in mind that Windows is definitely not a Unix-like OS, so you'd have to fix the script by providing the proper path to the <code>sh</code> executable on your system.
      </div>
4. Add a webhook by setting up a basic CI/CD process using [**Travis CI**](https://travis-ci.org) and [**AWS**](https://aws.amazon.com)
   1. Log in to [**AWS Console**](https://console.aws.amazon.com/), go to **S3** service, create two buckets for **staging** and **production** (choose unique names!), edit their public access settings (uncheck everything), go to bucket properties and enable static website hosting, then go to **My Security Credentials** and generate a new set of access keys. Phew! üòå
   <div class="bubble small subtle">
     <strong>S3 bucket names are globally unique.</strong> This means that if someone else has a bucket of a certain name, you cannot have a bucket with that same name.
   </div>

   2. Create **.travis.yml**:
       ```yaml
       language: node_js
       node_js:
         - "stable"
       script:
         - npm install --global workbox-cli
         - workbox generateSW workbox-config.js
       deploy:
         - provider: s3
           skip_cleanup: true
           access_key_id: $ACCESS_KEY_ID
           secret_access_key: $SECRET_ACCESS_KEY
           bucket: ________-staging
           region: eu-central-1
           acl: public_read
           on:
             branch: develop
         - provider: s3
           skip_cleanup: true
           access_key_id: $ACCESS_KEY_ID
           secret_access_key: $SECRET_ACCESS_KEY
           bucket: ________-production
           region: eu-central-1
           acl: public_read
           on:
             branch: master
       ```
5. Add your first integration by subscribing to [**Zeit Now**](https://zeit.co/now)
   1. Execute `sudo npm install --global --unsafe-perm now`
   2. Create **now.json**:
      ```json
      {
        "name": "homepage"
      }
      ```
   3. Run `now` to deploy the project
   4. Head over to **Zeit Now** Dashboard, delete the project, and try to deploy again using **Now for GitHub** this time
